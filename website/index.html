Folder-Agent-Binding

Right-click any folder in Windows → hand it to your AI agent as a standing context. It's how people already organize. Now your agent does too.

🆓 Free forever · Open source · Local-only · No account

What it does

You already think in folders — "this is the taxes stuff," "this is project X." Folder-Agent-Binding lets you tell your Hermes agent the same thing with one right-click. The agent gets a notification and a persistent context for that folder, so it understands the scope of its responsibility instead of staring at a flat pile of files.

This is the missing organizational layer for agent memory and RAG: folders become scoped, topic-based agent contexts that mirror real human structure.

🖱️ One right-click

Right-click a folder in Explorer → "Assign to Agent" → pick Goodybot, Dottie, Trader, or any profile.

🔁 Already-assigned warning

The dialog tells you if the folder is already bound to an agent, and lets you Assign or Unassign on the spot.

📨 Agent gets notified

Over its existing encrypted Telegram channel — no new account, no new transport.

🧠 Standing context

The agent now knows that folder is "its" area, with a structural summary of contents.

How it works

Explorer right-click→ assign_to_agent.exe (picks agent)→ POST 127.0.0.1:8771→ broker.py (saves binding)→ hermes send → Telegram → bot

No arrow leaves your machine except the single, user-initiated Telegram message. The broker binds to 127.0.0.1 only — it is not reachable from the network.

Install (Windows 11, elevated PowerShell)

# from the repo root:
.\install_windows.ps1

That registers the context-menu entry, starts the broker at logon, and drops a default agents.json. Edit %LOCALAPPDATA%\FolderAgentBinding\agents.json to set your agent names.
