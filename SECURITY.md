# SECURITY.md — Independent Validation Evidence

This project's safety claim is: **local-only, no secret handling, no file
exfiltration, loopback-only network.** We back that claim with automated,
reproducible third-party tooling. Results below are generated by running the
exact commands in CI (see `.github/workflows/validate.yml`) on every change.

## How to reproduce locally

```bash
# 1. Static analysis
python -m pip install bandit
bandit -r folder_agent_binding/ -f txt

# 2. Dependency CVE scan (runtime deps: NONE — stdlib only)
python -m pip install pip-audit
pip-audit -r requirements.txt   # empty by design

# 3. Prove zero runtime dependencies (the real SBOM)
python - <<'PY'
import ast, sys, pkgutil
stdlib = {m.name for m in pkgutil.iter_modules()} | set(sys.stdlib_module_names)
third = set()
for path in ("folder_agent_binding/broker.py",
             "folder_agent_binding/assign_to_agent.py"):
    tree = ast.parse(open(path).read())
    for node in ast.walk(tree):
        if isinstance(node, ast.Import):
            for n in node.names: third.add(n.name.split('.')[0])
        elif isinstance(node, ast.ImportFrom):
            if node.level == 0 and node.module: third.add(node.module.split('.')[0])
non_std = {m for m in third if m not in stdlib}
assert not non_std, f"third-party runtime imports: {non_std}"
print("OK: stdlib only, zero third-party runtime deps")
PY

# 4. Confirm loopback-only exposure (from a SECOND machine on the LAN)
nmap -p 8771 <windows-ip>      # expected: filtered/closed
```

## Source-first: no prebuilt binaries

This tool ships as **readable Python source**. You install by running the `.py`
files directly (`install_windows.ps1` copies them and runs
`pythonw broker.py serve`). There is nothing opaque to trust.

If you want standalone `.exe` wrappers for convenience, **build them yourself
from this source** — that way the binary is yours, produced locally, and
verifiable against the code you can read:

```bash
python build_windows.py            # builds broker.exe + assign_to_agent.exe, prints SHA256
python build_windows.py --update-security   # also writes the hashes into this file
```

Because everyone builds from the same audited source, the SHA256 you get is the
proof that *your* binary matches *this* code. We do not publish a binary for you
to download and wonder about — that would undercut the whole safety story.

### Why "third-party validation" and not just our word

The safety claims above are *checkable by anyone* with the free tools listed.
We do not ask you to trust us — we ask you to run `bandit` and `pip-audit` and
see zero findings yourself. That is the difference between "we say it's safe"
and "here is the proof it's safe."
